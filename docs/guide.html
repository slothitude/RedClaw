RedClaw User Guide

A minimal, provider-agnostic AI coding agent — v0.2.0

Installation

Windows (exe)

Download redclaw.exe from the latest release and run the installer.

Or install manually:

rem Copy to a directory on your PATH
copy redclaw.exe %LOCALAPPDATA%\RedClaw\

Linux / macOS

chmod +x install.sh
./install.sh

Creates a venv at ~/.redclaw/venv and symlinks the binary to ~/.local/bin/redclaw.

From Source

git clone https://github.com/slothitude/RedClaw.git
cd RedClaw
pip install -e .

Docker

docker compose up

Quick Start

Run redclaw with no arguments to see the interactive mode chooser:

RedClaw v0.2.0 — Choose a mode:

  1) REPL         Interactive CLI coding agent
  2) Dashboard    Web config GUI + process launcher (port 9090)
  3) WebChat      Browser-based chat (port 8080)
  4) Telegram     Telegram bot

  0) Exit
Tip: Pass --mode repl (or any mode) to skip the menu.

On first launch you'll need a provider and model. The default is zai/glm-5.1. To use OpenAI:

redclaw --provider openai --model gpt-4o

Modes

ModePortDescription
repl—Interactive CLI with streaming Markdown rendering, sessions, and slash commands
dashboard9090Flask web GUI for configuring providers, models, and launching other modes
webchat8080Browser-based chat with embedded HTML UI
telegram—Telegram bot with per-user sessions, file upload/download, and optional assistant mode
rpc—JSON-RPC over stdio (used by the Godot 4.6 GUI)

REPL

The default coding agent mode. Type natural language, and RedClaw uses tools to accomplish tasks. Supports streaming output with rich formatting.

redclaw --mode repl
> list all Python files in this project
> refactor the authentication module to use JWT
> /help

Dashboard

A web interface at http://127.0.0.1:9090 for configuring and launching RedClaw. PIN-protected config save/load.

redclaw --mode dashboard --dashboard-port 9090

WebChat

Browser-based chat with a built-in HTML UI. Connects via WebSocket for real-time streaming.

redclaw --mode webchat --port 8080

Telegram

Run as a Telegram bot. Each user gets their own session with isolated conversation history.

rem Set token via environment variable
set REDCLAW_TELEGRAM_TOKEN=your_bot_token
redclaw --mode telegram

With assistant mode enabled:

redclaw --mode telegram --assistant

Permissions

RedClaw has a 4-tier permission system controlling what tools can do:

ModeAllowed
askPrompts the user before executing write/bash commands
read_onlyOnly read_file, glob_search, grep_search
workspace_writeAll file operations + bash
danger_full_accessEverything, no prompts
Caution: danger_full_access lets the agent run any command without confirmation. Use only in trusted environments.

Memory

RedClaw has a persistent memory system that survives across sessions.

  • MEMORY.md and USER.md loaded once at session start (frozen snapshot)
  • Agent can store/recall/search memories via the memory tool
  • All content scanned for injection, exfiltration, and invisible unicode
  • Stored at ~/.redclaw/memory/

Subagents

Delegate tasks to isolated sub-agents with typed capabilities:

TypeToolsBest for
CODERcore + shellWriting and modifying code
SEARCHERcore + webResearch and information gathering
GENERALall non-excludedGeneral-purpose tasks

Features: depth limiting (max 2 levels), 5 tool rounds per subagent, 60s timeout, batch mode (up to 3 concurrent), retry-with-reflection (up to 3 retries).

Wisdom Inheritance (Crypt)

Subagents accumulate lessons in per-type "bloodline" files. Results are entombed for future learning. Cross-cutting patterns are stored in a living "dharma" document.

Skills

Skills are YAML+Python plugins the agent can create, view, and manage at runtime.

Discovery paths (in priority order):

  1. --skills-dir CLI flag
  2. <cwd>/skills/ (project-local)
  3. ~/.redclaw/skills/ (user-global)

Manifest formats:

  • SKILL.md — YAML frontmatter (--- delimited) + markdown body
  • skill.yaml — Pure YAML manifest

Agent tools: skills_list, skill_view, skill_manage (create/update/patch/delete)

Assistant Mode

Enabled with --assistant (Telegram mode only). Adds proactive personal assistant features:

  • Tasks — add, list, update, delete, search
  • Notes — CRUD + search
  • Reminders — schedule, check due, auto-notify
  • Persona — configurable name via assistant_config.json

Data stored at ~/.redclaw/assistant/ as JSONL files.

Knowledge Graph

Enabled with --knowledge. Uses Cognee for persistent knowledge graph memory.

ActionDescription
addStore facts in the graph
cognifyProcess stored data into graph structure
searchQuery the knowledge graph
memifySummarize graph insights to memory
pruneRemove old entries

Requires cognee package and a separate LLM API key (--knowledge-api-key). Data stored at ~/.redclaw/knowledge/.

Hooks

Run shell commands before or after tool execution:

  • Pre-hooks — receive REDCLAW_TOOL and REDCLAW_TOOL_INPUT env vars; non-zero exit blocks the tool
  • Post-hooks — receive REDCLAW_TOOL and REDCLAW_RESULT env vars
  • 30s timeout per hook execution

MCP Servers

Connect external tool servers via the Model Context Protocol (SSE transport):

redclaw --mcp-servers http://localhost:8006/sse http://localhost:8007/sse

Local servers included in servers/:

ServerPortDescription
TTS8006Text-to-speech (Coqui XTTS-v2 or edge-tts fallback)
STT8007Speech-to-text (Whisper base model)
Web Reader8003Headless browser page reading (Playwright)

Docker

rem Build and run
docker compose up

rem With custom config
docker compose up -e OPENAI_API_KEY=sk-...

Exposes WebChat (8080) and Dashboard (9090). Volumes for persistent data at /root/.redclaw.

Updating

When running as a standalone exe, RedClaw automatically checks for updates on startup (once per 24 hours).

  1. If a new version is found on GitHub Releases, you'll see a prompt
  2. Accept the update to download and install the new version
  3. Restart RedClaw to run the updated version

To manually update:

rem Download the latest redclaw.exe from GitHub Releases
rem Replace your existing exe
Linux: Run ./install.sh again — it will upgrade via pip.

Architecture

redclaw/
  api/            Provider-agnostic LLM client, SSE parser, provider registry
  runtime/        Conversation loop, session, compaction, permissions, hooks,
                  subagents (bloodlines, retry, crypt wisdom), prompt builder
  assistant/      Personal assistant: tasks, notes, reminders, scheduler, briefings
  memory_graph/   Cognee-backed knowledge graph memory
  tools/          Core tools, toolsets, memory, content scanning, assistant tools
  skills/         Skill discovery, loading, agent-managed CRUD, security scanner
  crypt/          Wisdom inheritance: bloodlines, entombment, dharma, metrics
  channels/       Abstract messaging layer (base + Telegram)
  mcp_client.py   MCP SSE client for external tool servers
  cli.py          REPL with rich rendering and interactive mode chooser
  updater.py      Auto-update via GitHub Releases
  rpc.py          JSON-RPC over stdio (Godot bridge)
  telegram_bot.py Telegram bot interface
  webchat.py      HTTP/WebSocket chat server
  dashboard.py    Flask config GUI and process launcher

servers/          Local MCP servers (TTS, STT, Web Reader)
godot/            Godot 4.6 GUI project
docs/             Documentation and user guide
